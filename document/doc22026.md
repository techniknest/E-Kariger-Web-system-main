# E-Karigar Web System Documentation (April 9, 2026)

## 1. Project Overview
E-Karigar is a full-stack service marketplace that connects skilled workers (vendors/karigars) with clients who need home services such as plumbing, electrical work, carpentry, and more. The platform supports three roles: CLIENT, VENDOR, and ADMIN.

## 2. Repository Structure
- e-karigar-backend: NestJS API server with Prisma ORM
- e-karigar-frontend: React + Vite frontend
- document: Documentation files

## 3. Tech Stack
Backend:
- NestJS (Node.js framework)
- Prisma ORM with PostgreSQL
- JWT authentication (passport-jwt)
- bcrypt for password hashing
- Cloudinary for image uploads

Frontend:
- React + Vite + TypeScript
- Tailwind CSS for styling
- React Router for routing
- React Query for server state
- Axios for API calls
- Framer Motion for animations

### 4.3 Modules and Responsibilities
1) Auth Module
- Controllers: auth.controller.ts
- Services: auth.service.ts
- Guard: auth.guard.ts
- DTOs: auth/dto/*

Key functionality:
- Register (always creates CLIENT role)
- Login (returns vendor status)
- Get current user
- Update profile

2) Admin Module
- Controllers: admin.controller.ts
- Services: admin.service.ts

Key functionality:
- Get pending and approved vendors
- Approve or reject vendor profiles
- Dashboard statistics
- Delete vendor or service

3) Vendors Module
- Controllers: vendors.controller.ts
- Services: vendors.service.ts

Key functionality:
- Vendor application workflow
- Vendor status checks
- Vendor profile fetch and update
- Updates user role to VENDOR on application

4) Services Module
- Controllers: services.controller.ts
- Services: services.service.ts
- Uses Cloudinary uploads

Key functionality:
- Public service listing
- Create/update/delete services (vendor-owned)
- Get service details with vendor reviews

5) Bookings Module
- Controllers: bookings.controller.ts
- Services: bookings.service.ts
- DTOs: bookings/dto/*

Key functionality:
- Create booking
- Vendor/client booking lists
- Update booking status
- Start job with OTP
- Revise quote (vendor)
- Approve or reject revised quote (client)

6) Reviews Module
- Controllers: reviews.controller.ts
- Services: reviews.service.ts
- DTOs: reviews/dto/*

Key functionality:
- Create review (completed bookings only)
- Vendor review listing and rating aggregate
- Vendor rating summary by user

7) Cloudinary Module
- Provider: cloudinary.provider.ts
- Service: cloudinary.service.ts

Key functionality:
- Upload image to Cloudinary

## 6. Key Functional Flows
### 6.1 Registration and Login
- Register endpoint creates a CLIENT user and returns JWT + user
- Login returns JWT + user object + vendor status
- Frontend stores token and user in localStorage

### 6.2 Vendor Application
- Client applies via /vendors/apply
- Creates VendorProfile with PENDING status
- User role updates to VENDOR
- Admin approves or rejects

### 6.3 Services Lifecycle
- Vendors create services with image upload
- Cloudinary stores images, API returns secure URL
- Services are visible in public listing

### 6.4 Booking Workflow
- Client books a service with date/time, address, problem description
- Vendor accepts/rejects (status change)
- Vendor starts job with OTP handshake
- Vendor can revise quote and request approval
- Client approves or rejects revised price

### 6.5 Reviews
- Client can review only completed bookings
- Vendor rating is aggregated in service details and vendor dashboard

## 9. Notes for New Contributors
- There are two login/register implementations (pages/auth vs pages root). App.tsx uses pages/auth.
- Booking flows use react-query polling for near-real-time updates.
- Vendor image uploads use Cloudinary with in-memory uploads.
- Admin vendor management and stats are in the admin module and vendor management page.
